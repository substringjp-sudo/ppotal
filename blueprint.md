# JPRail 프로젝트 청사진 (Blueprint)

## 1. 프로젝트 개요
JPRail은 일본 전역의 철도 데이터를 시각화하고 사용자에게 실시간 경로 탐색 및 여행 기록 서비스를 제공하는 차세대 철도 매니아용 플랫폼입니다.

### 핵심 시스템 가치
- **서버 권위적 아키텍처 (Server-Authoritative):** 대규모 철도 그래프 연산을 서버(Cloud Functions)에서 수행하여 클라이언트의 부담을 최소화.
- **안정적인 데이터 모델:** Firestore 기반의 정밀한 데이터 스키마를 통해 신뢰할 수 있는 구간/역 정보 제공.
- **프리미엄 인터랙션:** 'Snappy UX' 원칙에 따른 지연 없는 지도 조작 및 경로 시각화.

## 2. 현재 시스템 상태 (v0.2.0)

### 백엔드 (Firebase)
- **findPath (Cloud Functions):** 2GiB 메모리 환경에서 구동되는 최적화된 Dijkstra 경로 탐색 엔진.
- **그래프 모델:** 전체 노선 및 전이(Transfer) 엣지가 서버 상에서 실시간 캐싱되어 노선 간 환승 경로 탐색 지원.
- **Firestore:** 안정적으로 마이그레이션된 일본 전역의 역/구간 데이터셋 관리.

### 프론트엔드 (Next.js)
- **지능형 렌더링:** Leaflet Canvas 렌더러를 활용한 수천 개의 노선/역 정밀 렌더링.
- **여행 기록 엔진:** 드래그 앤 드롭 기반의 직관적인 이동 경로 기록 및 클라우드 동기화.
- **Snappy UX:** 줌/이동 시 의도적인 지연 및 투명도 전환을 제거한 즉각적인 반응성 확보.

## 3. 작업 히스토리 및 패치노트

### [v0.2.0] - 2026-04-19 (현재 버전)
#### 🚀 기능 개선 (Architecture)
- **서버 사이드 경로 탐색 최적화:** Cloud Functions 리소스 보강(2GiB) 및 서버 측 그래프 구성 로직 복구 (환승 노드 연결 성공).
- **멀티 노선 환승 지원:** 서버의 경로 탐색 엔진이 이제 서로 다른 철도 회사의 노선 간 환승 경로를 유기적으로 연결합니다.
- **SEO 최적화:** Next.js `metadataBase` 설정 적용으로 공유 이미지 및 메타데이터 정합성 확보.

#### ✨ UX/UI 고도화 (Snappy UX)
- **지연 시간 제거:** 지도 줌 변화 시 발생하던 300ms의 인위적인 대기 시간(LOD 가림 효과)을 제거하여 반응성 향상.
- **시각적 안정성:** 지도 조작 시 역과 노선이 사라지거나 흐려지던 효과(Flickering/Fading)를 제거하여 선명한 화면 유지.
- **렌더링 키 개선:** 여행 기록 삭제 시 지도의 레이어가 즉시 갱신되지 않던 문제 해결 (UsedSection Hash Key 적용).

#### 🐞 버그 수정 (Stability)
- **유령 지시선 박멸:** 드래그 종료 시 비동기 응답 처리 로직을 동기적으로 무효화(Ref-based Guard)하여 파란색 선이 남던 버그 해결.
- **경로 왜곡 방지:** 환승역에서 경로가 직선으로 튀어 보이던 현상을 방지하기 위해 렌더링 물리 세그먼트 분리화.
- **비정상 경로 지우기:** 비동기 경로 계산 중 드래그가 종료될 경우 프리뷰 상태(`draftTrip`)를 강제 초기화하도록 보강.

### [v0.1.0] - 지능형 아키텍처 기초 수립
#### 🏗️ 인프라 및 아키텍처 개편
- **Firestore 데이터 마이그레이션:** 레거시 대용량 JSON 기반 데이터 체계를 Firestore(Cloud DB)로 완전 마이그레이션하여 데이터 정합성 및 실시간성 확보.
- **라우팅 엔진 서버 이관:** 브라우저 성능을 저하시켰던 Dijkstra 최단 경로 연산 로직을 Firebase Cloud Functions(`findPath`)로 성공적으로 이관.
- **동적 로딩 시스템:** 앱 시작 시 모든 노선 데이터를 받던 방식에서, 필요한 노선 정보만 API로 요청하는 On-demand 방식 도입.
- **클라우드 연동 기반 구축:** 여행 기록의 서버 저장 및 사용자별 동기화를 위한 Firebase Auth & Firestore 인터페이스 구현.
